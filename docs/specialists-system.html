Motivação

Por que criar specialists?

O chat existente usava um único TriageAgent para tudo — triagem, dúvidas sobre documentos, pedidos de relatório, perguntas sobre materiais. Um agente generalista responde mal a perguntas especializadas, pois não tem contexto nem prompts otimizados para cada domínio.

Com o sistema de specialists, cada tipo de pergunta é atendido por um agente com: system prompt otimizado, ferramentas específicas (tool-use, RAG) e saída estruturada que a UI renderiza de forma rica.

⚡

Auto-detecção de intenção
Sem configuração — o orquestrador detecta o specialist certo por keywords.

🔌

Interface plugável
Novo specialist = implementar SpecialistPlugin. Sem tocar no chat.

🔒

Sem soberania de IA
Specialists só sugerem. State machine e rule engine continuam soberanos.

Os 5 Specialists

Cada especialista, seu domínio

Todos implementam a mesma interface SpecialistPlugin e suportam streaming nativo.

💬
TriageSpecialist
id: "triage"
Wrapper do TriageAgent existente. Conduz a triagem conversacional, coleta o escopo técnico via tool-use (submit_scope) e dispara a classificação determinística. É o fallback — ativa quando nenhum outro specialist bate.
Fallback padrão Tool-use nativo Streaming SSE ClassifyScopeUseCase
📄
DocumentSpecialist
id: "document"
Orienta sobre documentos necessários, status de cada arquivo enviado, como corrigir pendências e como obter ART/RRT. Recebe no contexto todos os documentos do caso com seus extractedData e pendencies.
20 keywords Contexto de docs Pendências Streaming
📊
ReportSpecialist
id: "report"
Gera relatórios diretamente pelo chat: Memorial Descritivo, Cronograma, Parecer Técnico, Proposta Comercial, Ordem de Serviço. Invoca GenerateReportUseCase com enrichWithAI: true e retorna metadata.reportId para a UI abrir o DocumentViewer.
6 tipos de relatório GenerateReportUseCase metadata.reportId DocumentViewer
🔨
MaterialsSpecialist
id: "materials"
Sugere materiais técnicos com especificações ABNT para os serviços da reforma. Usa RAG via pgvector: busca trechos de normas NBR relevantes antes de chamar o LLM. Retorna metadata.sources com as normas citadas.
RAG pgvector NormSearchService normas NBR metadata.sources
⚙️
ProcessSpecialist
id: "process"
Gera a sequência de etapas da obra com dependências, durações e observações críticas. Usa tool-use (submit_process_plan) para output estruturado. Injeta regras de domínio no prompt (impermeabilização antes de cerâmica, etc.). Retorna metadata.processSteps que a UI renderiza como tabela.
Tool-use submit_process_plan Regras de domínio metadata.processSteps

Código

Interface SpecialistPlugin

Implementar esta interface é suficiente para adicionar um novo specialist ao sistema.

// domain/specialists/SpecialistPlugin.ts

export interface SpecialistPlugin {
  readonly id: string          // "materials", "process", etc.
  readonly name: string        // exibido na UI
  readonly description: string
  readonly icon: string        // nome do ícone no design system
  readonly color: string       // token de cor do design system

  /** Retorna true se este specialist deve ser ativado automaticamente.
   *  Deve ser SÍNCRONO e leve — sem chamadas a LLM ou banco. */
  matchesIntent(message: string, ctx: PluginContext): boolean

  /** Processamento completo não-streaming */
  process(ctx: PluginContext): Promise<PluginResult>

  /** Streaming: textChunks emite pedaços; result resolve ao final com metadata */
  processStream(ctx: PluginContext): {
    textChunks: AsyncIterable<string>
    result: Promise<PluginResult>
  }
}

export interface PluginResult {
  text: string
  metadata: {
    specialistId: string
    reportId?: string         // ReportSpecialist
    sources?: NormSource[]    // MaterialsSpecialist
    processSteps?: Step[]     // ProcessSpecialist
  }
}
💡

Para adicionar um novo specialist, basta criar um arquivo em application/specialists/, implementar a interface, e registrá-lo em ChatOrchestrator.ts na função createOrchestrator(). Nenhum outro arquivo precisa ser alterado.

Roadmap

Extensões futuras possíveis

🏛️

NormsSpecialist

Consulta focada a normas ABNT, código de obras municipal, leis de condomínio. RAG mais especializado com citações precisas.

💰

BudgetSpecialist

Estimativa de custo da reforma com base no escopo, cidade e índice CUB. Cálculo de BDI e margem para o parceiro.

🏗️

StructuralSpecialist

Para casos com impacto estrutural/prumadas (CRITICAL). Orienta sobre laudo estrutural, AVCB, licenças municipais.

🤝

PartnerSpecialist

Explica o processo de atribuição de parceiro, SLA, vistorias agendadas e como se comunicar com o responsável técnico.

🔄

Intent via LLM

Substituir keyword-matching por chamada leve ao LLM (claude-haiku) para detecção de intenção mais robusta e multilíngue.

🔌

Specialists por Tenant

Configurar quais specialists estão disponíveis por tenant. Administradoras podem habilitar/desabilitar pelo painel de superadmin.
